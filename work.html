Portfolio

Portfolio

The portfolio project had three stages of development throughout this semester; first I had to dream it in XD, then build it with html and CSS and finally redesign the portfolio based on user tests and general feedback.

1 - The portfolio dream in XD

The assignment was to create a portfolio prototype in XD. My portfolio prototype was experience-based, which would challenge me and a potential user if I chose to code it. Because of the experience-based design and the interface, a user would easily loose the general overview of the site and “get lost”. Furthermore, I would be challenged when I had to add new projects to my portfolio later.

2 - Portfolio website with html and CSS

At this stage I had to code the portfolio from scratch. Since the prototype in XD was experience-based and would make a user "get lost" on the site. I had to rethink the design and the interface and I decided to skip the original ideas and start all over. In the second portfolio version the interface was still a mess and a user could easily get lost when I added new projects. Some of the fails was due to a bad design and others were simply there because I did not know how to code them properly. Furthermore, I had not yet learned how to make a website responsive and the result was a website that only worked on desktops. Back to improve the design and the code.

3 - The exam and redesign of the portfolio

At this stage in the learning process I knew that it was easier for me to approach a project when I worked from the five stages of Design Thinking; empathize, define, ideate, prototype and test. I also created a Gantt chart in Excel to keep track of my project time and uploaded the portfolio code in GitHub

Empathize and Define

First I had to be clear of what I want my portfolio to be? Playful, serious, avantgarde? How do I want to portray myself for a potential employer, a collegue or a fellow student?

One of the things I really wanted to show in my portfolio is the fact that I like to be creative. In the first stage of my portfolio I incorporated a handdrawn bird as a background image and I thought that it would be a nice way to show my creative side, so I reused that part of the design.
Since one of my potential users would be an employer I also wanted my portfolio to demonstrate the serious side of me; that I'm stable, trusworthy and loyal. So I decided to go with the classic blue colors to symbolize these types of virtues.

Ideate

To create a clear workflow for myself I segregated the ideate process in two; the informative structure and the design.

Structure

In the structural process I started with an analogue sitemap where I chose the classic hierarchical approach. The wireframe should be as simple as possible - not just for the usability but also to make the work process as manageable as possible the short working time taken into account. Therefore I decided to a maximum of three basic pages to the wireframes. The advantage of this decision was that it saved me some time in the coding process and made it easier for me to be consistent with the design on all sites of the portfolio.

Design

In the design process I used both analogue and digital sketches to create moodboard, mockups and styletyles for my portfolio.


Dual page design

As you can see from the mockups I worked with a dual page design with two boxes on the left and a gallery on the right. The benefit of this design is that I can use it for both the project- and the learning page. It creates an overview of all the projects/learning processes and gives a potential user to decide what specific project he/she would like to see. In the two previous portfolios the projects were simply stacked in a one-paged site where you had to scroll a lot to see the individual projects.


In terms of the gallery features I also played with the idea of a slide function. The problem with this type of gallery is that the user cannot see all the projects at once.


The fontpage I wanted to be different than the content pages.

I chose to go with the mockup that had a box that was off-set compared to the dual page design. It made more sence to me in terms of sticking to the design idea and keep the vertical and horisontal lines. You can argue that a rotated box would make the front page stand out, but I simply liked the off-set box better.

Colors

I wanted my portfolio to convey both a playful and a serious side. To convey the playful side of me I went with a handdrawn image that I edited in photoshop. The image is originally just a b/w pencil drawing but in photoshop I embedded the image and added the blue color I wanted.

Fonts

The font was difficult and I played with many different types. I chose to go with the font Palanquin, not just because of the readability but because I liked its clean style and the variations between the lighter and the bolder font.


Prototype

After working with the informative structure and design I now had a solid idea and fundament for continuing to the next stage - the XD prototype.


Test

Since my mayor challenge was to create a website that wasn't too experiment-based, I wanted to check whether a user would be able to navigate on the site without getting lost. I wanted to see the users actually clicking around the site, so I went with the qualitative "think-aloud" usertest. I asked three persons between 17 and 44, two girls and a guy, to click around my portfolio and give me feedback. After they had clicked around by themselves I asked them to individually to find my Dogwalk Project. All but one was able find it. I thought the succes rate 2/3 was pretty good. I asked the girl that couldn't find the project what I could do differently in order for her to find it, and she thought I should maybe do the font in a diferent color, or just bigger. I made the font a little bigger but kept the white textcolor. The overall navigation stayed the same.

Dogwalk App

Dogwalk App

The challenge

To develop an app prototype based on one of the following themes; wheather, listing alerts or sportstracking.
I decided to go with the third option and create a sportstracking app for dogs.

The process

In the process of developing a prototype in XD, I worked from the concept of Design Thinking and split the tasks in five categories

  • Empathize
  • Define
  • Ideate
  • Prototype
  • Test

To see the original prototype and a full description of the work process.

Remake of the project

Based on the feedback I got from the prototype, I chose to redesign the search page of the app. The interface of the original search page could be more user friendly. To do so, I moved the search icon into the actual search field and changed the background and font color of the call-to-action button. The difference is clear on the images below.

With a few changes the interface is optimized and the search page is more user friendly. Even though I sometimes get frustrated about making errors in the learning process, I am also surprised how little you have to change to make a site better.
